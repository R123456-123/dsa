-> A binary treee is a hierarchical structure where each node at most have left child or right child, not necessarily sorted, not necessarily complete
-> root - topmost node, leaf - node with no children(if the root has no children then it is a leaf node), height - longest path from node to leaf (height of leaf = 0, empty tree is -1), depth - distance from root, subtree - tree formed by any node and its children, sibling - nodes with same parent
-> height - ©(2) - ©(1) - ©(1) -- ©(0) -- ©(0) -- ©(0) -- ©(0) (height of root = height of root)
-> Binary Tree Properties
Maximum number of nodes at level i = 2^i
Maximum number of nodes in a tree of height h = 2^(h+1) – 1
Minimum height for N nodes = ceil(log₂(N + 1)) – 1
Maximum height for N nodes = N – 1 (skewed tree)

-> Types of Binary Trees
1. Full Binary Tree, Every node has 0 or 2 children.
2. Complete Binary Tree, All levels filled except possibly last, and last filled left to right, Used for Heaps.
3. Perfect Binary Tree, All levels fully filled, Nodes = 2^(h+1) − 1.
4. Balanced Binary Tree, Height = O(log N), Examples: AVL Tree, Red-Black Tree, Segment Tree
5. Skewed Binary Tree, All nodes only to left or right. Height = N-1.
6. Binary Search Tree (BST), Left < Root < Right, Used for sorted operations.

-> Tree Traversals 
A. DFS Traversals
1. Inorder (L, Root, R), Gives sorted output for BST.
Use: Validate BST, Convert tree to sorted list

2. Preorder (Root, L, R)
Use: Construct tree, Serialize tree

3. Postorder (L, R, Root)
Use: Delete tree, Calculate height, diameter, subtree sum

B. BFS Traversal, Level Order Traversal, Use Queue.
Use: Shortest path to leaf, Right side view, left view, Zigzag traversal, Level by level printing
